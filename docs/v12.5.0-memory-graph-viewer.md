# v12.5.0 - Memory graph visualization

## What is included

- Adds a local Cytoscape.js memory graph viewer export tool.
- Exports graph nodes, edges, type counts, weights, and linked knowledge summaries into a static `graph.json`.
- Ships browser assets for both 2D and 3D graph exploration.
- Reads the memory database in SQLite read-only mode and writes only to the chosen export directory.
- Keeps real memory data out of the repository and release artifacts.

## Who should try it

Use this release if you want to inspect how concepts, projects, facts, lessons, and knowledge records connect inside a Total Agent Memory database without uploading private memory content anywhere.

## How to use

```bash
python src/tools/memory_graph_viewer_export.py --output /tmp/tam-graph-viewer
```

Then open `index.html` in the output directory. Use `--db`, `--max-nodes`, `--max-edges`, and `--min-edge-weight` when you want to inspect a specific database or produce a smaller graph.

## Privacy boundary

This release contains only system files: source code, tests, scripts, docs, and static viewer assets. It does not include live memory databases, WAL/SHM files, logs, transcripts, raw captures, backups, or exported private memory content.

## Current limitations

- The viewer is a local static export, not a hosted service.
- Large graphs may need `--max-nodes` or `--min-edge-weight` for smoother browser rendering.
- The exported `graph.json` can contain summaries from the selected local memory database, so keep generated viewer output private unless you have reviewed it.
